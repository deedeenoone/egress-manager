功能特性
═══════════════════════════════════════════════════════════════════════════════

✅ 核心功能
  • 为任意systemd服务配置独立的出网IP和路由策略
  • 支持按入站端口绑定不同出站IP
  • 支持多协议混跑（Snell、SS、SOCKS等）
  • 支持多网卡多IP场景
  • 动态修改配置，无需系统重启

✅ 技术特性
  • 基于cgroup v2进程隔离
  • 使用iptables fwmark进行流量标记
  • 自定义路由表管理（700-899）
  • 支持POSTROUTING SNAT改源IP
  • 通过connmark保护已建立连接

✅ 管理特性
  • 一键初始化系统
  • 简洁的命令行界面
  • 自动生成systemd drop-in
  • 完整的故障排查工具
  • 实时查看路由表和规则

命令参考
═══════════════════════════════════════════════════════════════════════════════

初始化系统 (仅需一次):
  sudo bash egress-manager.sh init

配置服务出网策略:
  sudo bash egress-manager.sh set <service> <iface> <gateway> <subnet> [srcip]
  
  示例:
    sudo bash egress-manager.sh set snell-443 eth0 10.0.0.1 10.0.0.0/24 10.0.0.5

删除配置:
  sudo bash egress-manager.sh remove <service>

查看配置:
  sudo bash egress-manager.sh show <service>

列表显示:
  sudo bash egress-manager.sh list

查看路由表:
  sudo bash egress-manager.sh routes

查看规则:
  sudo bash egress-manager.sh rules

帮助:
  bash egress-manager.sh help

使用场景
═══════════════════════════════════════════════════════════════════════════════

场景1: VPS多IP，按入站端口使用不同出站IP
  描述: 单个VPS分配多个公网IP，需要不同的服务实例从不同IP出
  应用: 多入站端口Snell、负载均衡、IP轮转

场景2: 多协议多网卡隔离
  描述: 多个协议（Snell、SS、SOCKS等）跑在不同网卡和IP上
  应用: 基础设施隔离、不同服务线路划分

场景3: 出口IP轮转和故障转移
  描述: 根据业务需求动态调整出口IP
  应用: ISP故障转移、流量均衡、地域路由

场景4: 多租户隔离
  描述: 为不同租户配置独立的出网IP和路由策略
  应用: 代理管理平台、多客户制系统
